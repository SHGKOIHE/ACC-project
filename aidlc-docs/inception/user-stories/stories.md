## EP-01: 회원 인증

> **As a** 신규 사용자로서  
> **I want to** 앱에 회원가입하고 로그인할 수 있기를  
> **So that** 공동 구매 방을 개설하거나 참여할 수 있다

**인수 기준 (Given-When-Then)**

```
Scenario: 이메일로 회원가입
  Given 앱을 처음 실행한 사용자가
  When 이메일, 비밀번호, 닉네임을 입력하고 가입 버튼을 누르면
  Then 계정이 생성되고 홈 화면으로 이동한다

Scenario: 로그인
  Given 가입된 계정이 있는 사용자가
  When 이메일과 비밀번호를 입력하면
  Then JWT 토큰이 발급되고 홈 화면으로 이동한다

Scenario: 로그아웃
  Given 로그인 상태의 사용자가
  When 로그아웃을 선택하면
  Then 세션이 종료되고 토큰이 무효화된다
```

**관련 페르소나**: 김민준, 이지은, 박승현, 최하은

## EP-02: 방 개설 (공동 배달)

> **As a** 방장으로서  
> **I want to** 공동 배달 주문 방을 개설할 수 있기를  
> **So that** 근처 사람들과 배달비를 나눠 내고 음식을 주문할 수 있다

**인수 기준 (Given-When-Then)**

```
Scenario: 공동 배달 방 개설
  Given 로그인한 사용자가
  When 방 유형을 "공동 배달"로 선택하고
       식당 정보, 배달 주소, 최대 인원, 마감 시간을 입력하면
  Then 방이 생성되고 지도에 표시되며 참여자 모집이 시작된다

Scenario: 배달비 분담 미리 보기
  Given 방을 개설하는 방장이
  When 예상 배달비와 최대 인원을 입력하면
  Then 1인당 예상 배달비(배달비 ÷ 최대 인원, 올림)가 표시된다
```

**관련 페르소나**: 김민준

## EP-03: 방 개설 (모임 배달 / 직접 외식)

> **As a** 방장으로서  
> **I want to** "배달 시켜서 함께 먹기" 또는 "직접 식당 가기" 유형의 방을 개설할 수 있기를  
> **So that** 배달 외에도 다양한 식사 모임을 구성할 수 있다

**인수 기준 (체크리스트)**

- [ ] 방 개설 시 유형 선택 가능: `공동 배달` / `배달+모여먹기` / `직접 외식`
- [ ] `배달+모여먹기`: 모일 장소 주소 입력 필드 추가 표시
- [ ] `직접 외식`: 방문할 식당 주소, 예상 출발 시간 입력 필드 표시
- [ ] 방 유형이 지도 탐색 화면에서 아이콘으로 구분되어 표시됨
- [ ] 방 상세 화면에서 모임 유형과 장소 정보가 명확히 표시됨

**관련 페르소나**: 이지은

## EP-04: 방 탐색

> **As a** 참여자로서  
> **I want to** 지도에서 근처 열린 방을 탐색하고 필터링할 수 있기를  
> **So that** 내 위치와 음식 취향에 맞는 방을 빠르게 찾을 수 있다

**인수 기준 (Given-When-Then)**

```
Scenario: 지도 기반 방 탐색
  Given 로그인한 사용자가 탐색 화면을 열면
  When 현재 위치 주변의 모집 중인 방들이 지도에 마커로 표시되고
  Then 마커를 탭하면 방 이름, 식당, 모임 유형, 남은 자리, 마감 시간이 표시된다

Scenario: 식당 카테고리 필터
  Given 탐색 화면에서
  When 사용자가 음식 카테고리(한식, 중식, 분식 등)를 선택하면
  Then 해당 카테고리의 방만 지도에 표시된다
```

**관련 페르소나**: 박승현, 최하은

## EP-05: 방 참여 및 메뉴 선택

> **As a** 참여자로서  
> **I want to** 방에 참여하고 내 메뉴를 선택할 수 있기를  
> **So that** 공동 주문에 내 주문이 포함되도록 할 수 있다

**인수 기준 (Given-When-Then)**

```
Scenario: 방 참여
  Given 모집 중인 방의 상세 화면에서
  When 사용자가 "참여하기" 버튼을 누르면
  Then 참여자 목록에 추가되고 채팅방과 주문 화면에 접근할 수 있다

Scenario: 메뉴 입력 (직접 외식 / 배달)
  Given 방에 참여한 사용자가
  When 주문 화면에서 메뉴명과 수량, 가격을 입력하거나 선택하면
  Then 전체 참여자의 주문 목록에 해당 항목이 추가된다

Scenario: 배달비 분담 확인
  Given 주문 화면에서
  When 현재 참여자 수가 업데이트되면
  Then 1인당 배달비(총 배달비 ÷ 실제 참여자 수, 소수점 올림)가 자동 갱신된다
```

**관련 페르소나**: 박승현

## EP-06: AI 음식 추천

> **As a** 메뉴를 정하지 못한 참여자로서  
> **I want to** AI로부터 음식 추천을 받을 수 있기를  
> **So that** 빠르게 메뉴를 결정하고 맞는 방을 찾거나 직접 개설할 수 있다

**인수 기준 (Given-When-Then)**

```
Scenario: AI 추천 요청
  Given 로그인한 사용자가 AI 추천 화면으로 이동하면
  When 선호 음식 카테고리, 가격대, 현재 위치를 입력하고 추천 요청을 하면
  Then AI가 주변 식당 기반으로 1~3개 음식/식당을 추천한다

Scenario: 추천 결과로 방 탐색
  Given AI 추천 결과 화면에서
  When 사용자가 추천 식당 항목을 탭하면
  Then 해당 식당의 열린 방 목록으로 이동하거나
       방이 없을 경우 "방 개설하기" 버튼이 표시된다
```

**관련 페르소나**: 최하은

## EP-07: 실시간 채팅

> **As a** 방 참여자로서  
> **I want to** 방 내에서 다른 참여자와 실시간으로 채팅할 수 있기를  
> **So that** 주문 관련 소통을 앱 안에서 원활하게 할 수 있다

**인수 기준 (체크리스트)**

- [ ] 방에 참여한 사람만 채팅방에 접근 가능
- [ ] 메시지가 모든 참여자에게 실시간(500ms 이내)으로 전달됨
- [ ] 시스템 메시지: 참여자 입장/퇴장, 방 상태 변경 자동 표시
- [ ] 메시지에 작성자 닉네임과 시간 표시
- [ ] 방장이 방을 종료하면 채팅이 읽기 전용으로 전환됨

**관련 페르소나**: 김민준, 이지은, 박승현, 최하은

## EP-08: 주문 확정 및 배달비 정산

> **As a** 방장으로서  
> **I want to** 전체 주문을 확정하고 배달비 정산 내역을 공유할 수 있기를  
> **So that** 모든 참여자가 얼마를 내야 하는지 명확히 알 수 있다

**인수 기준 (Given-When-Then)**

```
Scenario: 주문 확정
  Given 방장이 모든 참여자 주문이 완료됐다고 판단하고
  When "주문 확정" 버튼을 누르면
  Then 방 상태가 "모집중"에서 "주문완료"로 변경되고
       모든 참여자에게 FCM 푸시 알림이 전송된다

Scenario: 최종 정산 내역 확인
  Given 주문이 확정된 방에서
  When 참여자가 정산 화면을 열면
  Then 자신의 주문 항목 합계 + 1인당 배달비(소수점 올림)가 표시된다

Scenario: 배달 완료 처리
  Given 방 상태가 "배달중"인 상황에서
  When 방장이 "배달 완료" 버튼을 누르면
  Then 방 상태가 "완료"로 변경되고 참여자에게 알림이 전송된다
```

**관련 페르소나**: 김민준, 이지은

## EP-09: 식당 정보 연동

> **As a** 방장으로서  
> **I want to** 식당 메뉴와 가격 정보를 앱에 쉽게 불러올 수 있기를  
> **So that** 참여자들이 메뉴와 가격을 확인하고 주문할 수 있다

**인수 기준 (체크리스트)**

- [ ] **Primary (배달앱 API)**: 배달의민족 또는 쿠팡이츠 API가 지원되는 경우, 링크 입력만으로 메뉴/가격/배달비 자동 불러오기
- [ ] **Fallback (링크 공유)**: API 미지원 시 방장이 배달앱 링크를 방에 공유, 참여자가 직접 확인 후 메뉴명·가격 수동 입력
- [ ] 방장이 식당 이름, 최소 주문금액, 배달비를 직접 입력하는 수동 입력 옵션도 항상 제공
- [ ] 불러온 메뉴 목록에서 참여자가 자신의 메뉴를 탭하여 선택 가능

**관련 페르소나**: 김민준, 이지은

## EP-10: 관리자 운영

> **As a** 관리자로서  
> **I want to** 앱 운영 현황을 모니터링하고 신고를 처리할 수 있기를  
> **So that** 플랫폼이 안전하고 건강하게 운영된다

**인수 기준 (체크리스트)**

- [ ] 관리자 전용 대시보드 접근 (일반 사용자 접근 불가)
- [ ] 사용자 목록 조회, 계정 정지/해제
- [ ] 신고된 방/사용자 목록 조회 및 처리 (경고/정지)
- [ ] 주요 통계 확인: 일별 활성 방 수, 신규 가입자 수

**관련 페르소나**: 서비스 운영팀
