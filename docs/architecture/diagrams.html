3. WebSocket / FCM 연동 구조

STOMP 기반 실시간 채팅과 FCM 푸시 알림 발송 흐름.

sequenceDiagram
    participant App as React Native
    participant Tunnel as Cloudflare Tunnel
    participant WS as Spring Boot
(STOMP)
    participant Redis as Redis
(Pub/Sub)
    participant FCM as Firebase FCM

    Note over App, WS: WebSocket 연결
    App->>Tunnel: WS 연결 요청
    Tunnel->>WS: STOMP CONNECT

    Note over App, FCM: 채팅 메시지 전송
    App->>WS: /app/room/{roomId}/chat
    WS->>Redis: Publish (채팅 이벤트)
    WS-->>App: /topic/room/{roomId}/chat (실시간)
    WS->>FCM: 백그라운드 사용자에게 푸시
    FCM-->>App: 푸시 알림 수신

    Note over App, FCM: 방 참여 알림
    App->>WS: POST /api/rooms/{id}/join
    WS->>Redis: INCR room:{id}:count
    WS-->>App: /topic/room/{roomId}/members
    WS->>FCM: "새 참여자 입장" 알림
    FCM-->>App: 푸시 알림 수신

4. AI 추천 파이프라인

규칙 엔진 필터링 -> Lambda(Gemini Flash) 설명 생성의 2단계 파이프라인.

sequenceDiagram
    participant App as React Native
    participant SB as Spring Boot
    participant Rule as Rule Engine
(Spring Boot 내부)
    participant APIGW as API Gateway
    participant Lambda as Lambda
    participant Gemini as Gemini Flash

    App->>SB: POST /api/ai/recommend
{categories, maxPrice, mood}
    SB->>Rule: 필터링 요청

    Note over Rule: 1단계: 규칙 기반 필터 + 랭킹
카테고리 매칭
거리 필터 (Haversine)
가격대 필터
참여율/마감시간 랭킹

    Rule-->>SB: Top 3~5 식당

    SB->>APIGW: POST /recommend
{restaurants, user_preferences}
    APIGW->>Lambda: Invoke
    Lambda->>Gemini: 설명 생성 요청
"왜 이 식당을 추천하는지 2줄"

    Note over Gemini: 2단계: 자연어 설명 생성

    Gemini-->>Lambda: 설명 텍스트
    Lambda-->>APIGW: {recommendations}
    APIGW-->>SB: 응답
    SB-->>App: 식당 목록 + AI 설명

5. ERD (Entity Relationship Diagram)

6개 핵심 테이블 관계도. members, rooms, room_participants, order_items, settlements, member_settlements.

erDiagram
    members {
        bigserial id PK
        varchar nickname UK
        varchar device_token UK
        timestamp created_at
    }

    rooms {
        bigserial id PK
        bigint host_id FK
        varchar title
        varchar meeting_type
        varchar restaurant_name
        varchar restaurant_address
        double latitude
        double longitude
        integer delivery_fee
        integer max_participants
        varchar status
        timestamp closed_at
        varchar meeting_address
        varchar account_number "AES-256 암호화"
        varchar account_holder
        varchar bank_name
        timestamp created_at
        timestamp updated_at
        integer version "낙관적 락"
    }

    room_participants {
        bigserial id PK
        bigint room_id FK
        bigint member_id FK
        timestamp joined_at
    }

    order_items {
        bigserial id PK
        bigint room_id FK
        bigint member_id FK
        varchar menu_name
        integer quantity
        integer price
        timestamp created_at
    }

    settlements {
        bigserial id PK
        bigint room_id FK "UNIQUE"
        integer total_menu_amount
        integer total_delivery_fee
        integer participant_count
        integer fee_per_person "올림 계산"
        integer host_surplus
        timestamp created_at
    }

    member_settlements {
        bigserial id PK
        bigint settlement_id FK
        bigint member_id FK
        integer menu_amount
        integer delivery_fee_share
        integer total_amount
        boolean is_host
    }

    members ||--o{ rooms : "creates (host)"
    members ||--o{ room_participants : "joins"
    members ||--o{ order_items : "orders"
    members ||--o{ member_settlements : "settles"
    rooms ||--o{ room_participants : "has"
    rooms ||--o{ order_items : "contains"
    rooms ||--|| settlements : "produces"
    settlements ||--o{ member_settlements : "details"

6. 방 상태 전이 다이어그램

rooms.status 필드의 상태 머신. 5개 상태, 취소는 OPEN/CLOSED/CONFIRMED 모두에서 가능.

stateDiagram-v2
    [*] --> OPEN: 방 생성

    OPEN --> CLOSED: 방장 수동 마감
시간 자동 마감
    OPEN --> CANCELLED: 방장 취소
참여자 0명

    CLOSED --> CONFIRMED: 방장 주문 확정
    CLOSED --> CANCELLED: 방장 취소

    CONFIRMED --> COMPLETED: 방장 완료 처리
    CONFIRMED --> CANCELLED: 방장 취소
(식당 거부 등)

    COMPLETED --> [*]
    CANCELLED --> [*]
