**작성자**: ui (UI/UX 디자이너)
**작성일**: 2026-05-24
**대상**: 배달비 절약 음식 공동구매 앱 (React Native / Expo)
**타겟 사용자**: 학교 주변 대학생

    |           v

    |           |
    |           v
    |     [홈 화면으로 이동]

    +---> [홈 화면] (재방문) <-----------------+
                |                              |
    +-----------+------------+                 |

## 2. 주요 시나리오

### 시나리오 A: 최초 닉네임 등록

```
[앱 최초 실행]
    |
    v
[스플래시 화면]
  - 앱 로고 + 슬로건 표시
  - 로컬 스토리지에 닉네임 없음 확인
    |
    v
[닉네임 등록 화면]
  - 닉네임 입력 (2~10자, 한/영/숫자)
  - 학교 선택 (드롭다운 또는 검색)
  - "시작하기" 버튼
    |
    v (성공)
[홈 화면]
  - 방 목록 + 지도 기본 탭 표시
  - 온보딩 툴팁 (최초 1회): "지도에서 근처 방을 찾아보세요"
```

---

### 시나리오 B: 방 탐색

```
[홈 화면]
    |
    +--- [지도 탭 선택]
    |         |
    |         v
    |   [지도 화면]
    |     - 현재 위치 중심 지도 표시
    |     - 방 유형별 마커 표시:
    |       [D] 공동배달  [G] 배달+모여먹기  [R] 직접외식
    |     - 음식 카테고리 필터 칩 (한식 / 중식 / 분식 / 기타)
    |         |
    |         v (마커 탭)
    |   [방 미리보기 카드 (Bottom Sheet)]
    |     - 식당명 / 방 유형 뱃지 / 남은 자리 / 마감까지 시간
    |     - "상세 보기" 버튼
    |
    +--- [목록 탭 선택]
              |
              v
        [방 목록 화면]
          - RoomCard 리스트
          - 필터: 유형 / 카테고리 / 거리
          - 마감 임박 순 기본 정렬
```

---

### 시나리오 C: 방 참여

```
[방 상세 화면]
  - 식당 정보 / 배달 주소 / 모임 유형
  - 참여자 목록 (ParticipantList)
  - 현재 1인당 예상 배달비
  - 남은 자리 / 마감 시간
    |
    v (참여하기 버튼 탭)
[참여 확인 팝업]
  - "예상 배달비: N원 / 참여 시 함께 분담됩니다"
  - 확인 / 취소
    |
    v (확인)
[메뉴 선택 화면으로 이동]
```

---

### 시나리오 D: 메뉴 선택

```
[메뉴 선택 화면]
    |
    +--- [식당 메뉴 API 불러오기 성공]
    |         v
    |   [메뉴 리스트 표시]
    |     - OrderItemRow 컴포넌트 리스트
    |     - 탭하여 수량 조절 (+ / -)
    |
    +--- [API 실패 / 수동 입력]
              v
        [메뉴 직접 입력 폼]
          - 메뉴명 / 수량 / 가격 입력
    |
    v (공통)
[주문 내역 확인]
  - 내 주문 항목 요약
  - 예상 배달비 (현재 참여자 수 기준 자동 계산)
  - "주문 완료" 버튼
    |
    v
[채팅 화면으로 이동]
  - 시스템 메시지: "{닉네임}님이 메뉴를 선택했습니다"
```

---

### 시나리오 E: 정산 확인

```
[방장: 주문 확정]
  - "주문 확정" 버튼 탭
  - 확인 팝업: "모든 참여자에게 확정 알림이 전송됩니다"
    |
    v
[전체 참여자: FCM 푸시 알림]
  - "{방 이름} 주문이 확정되었습니다"
    |
    v
[정산 화면]
  +----------------------------+
  | 내 주문 항목               |
  |   - 치킨마요 덮밥  8,500원 |
  |   - 단무지 추가      500원 |
  +----------------------------+
  | 1인당 배달비         1,000원 |
  +----------------------------+
  | 내가 낼 금액        10,000원 |
  +----------------------------+
  | 계좌: 카카오뱅크 1234-xxxx  |
  | (방장 계좌 표시)            |
  +----------------------------+
  [송금 완료 표시 버튼]
    |
    v
[배달 완료 후 방장이 "배달 완료" 처리]
    |
    v
[방 종료 / 평점 화면 (선택)]
```

## 3. 링크/QR 공유 참여 흐름 (배민 단체주문 참고)

```
[방장: 공유 버튼 탭]
    |
    v
[공유 시트]
  - 카카오톡 공유
  - 링크 복사
  - QR 코드 표시
    |
    v (수신자: 앱 없음)
[웹뷰 랜딩 페이지]
  - 방 정보 미리보기
  - "앱 다운로드" 버튼
  - (웹뷰) 메뉴 선택 기능 (제한)
    |
    v (수신자: 앱 있음)
[딥링크 → 방 상세 화면 바로 이동]
    |
    v
[시나리오 C: 방 참여로 합류]
```

## 4. 모임 유형별 분기 흐름

```
[방 개설 화면 — 유형 선택]
    |
    +---> [공동배달]
    |       - 배달 주소 입력 (필수)
    |       - 최소 주문금액 / 배달비 입력
    |       - 메뉴 선택 화면 → 장바구니 통합
    |
    +---> [배달+모여먹기]
    |       - 배달 주소 + 모일 장소 입력
    |       - 인원 제한 설정
    |       - 지도에 [G] 아이콘 마커
    |
    +---> [직접외식]
            - 식당 주소 + 예상 출발 시간 입력
            - 참가 확정 버튼 UX (EatWith 참고)
            - 지도에 [R] 아이콘 마커
```
